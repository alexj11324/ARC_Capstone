# SLOSH Dataset Feature Dictionary (Parquet Version)

本项目中的 SLOSH 数据集代表了 NOAA 的 **S**ea, **L**ake, and **O**verland **S**urges from **H**urricanes 模型结果。

**⚠️ 重要概念说明**：
1.  **非时序数据**：此数据集**不是**时间序列数据。它不包含洪水随时间变化的过程。
2.  **MOMs (Maximum of Maximums)**：这是一个**静态的风险评估数据集**。每个网格的数值代表了在该飓风等级下，模拟成千上万种不同路径、速度的飓风后，该位置**可能达到的最高水位**（最坏情况快照）。
3.  **海拔 vs 水深**：表中的数值是**水位海拔 (Surge Elevation)**，而非地面的积水深度。

## 1. 核心计算公式

在评估具体地点的灾害风险时，请务必使用以下逻辑：

> **实际淹没深度 (Inundation Depth) = 浪高 (Surge Elevation) - 地形高度 (Topography)**

*   **如果结果 > 0**：该区域被淹没，数值即为水深。
*   **如果结果 <= 0 或 Surge 为 NULL**：该区域未被淹没（干旱）。

## 2. 字段详细定义

### 2.1 基础识别与空间信息

| 字段名称 | 含义 | 类型 | 详细说明 |
| :--- | :--- | :--- | :--- |
| **basin** | 流域/盆地代码 | STRING | **分区字段**。标识模拟覆盖的地理区域（如 `ny3mom` 为纽约，`ap3mom` 为佛罗里达）。 |
| **poly_id** | 网格 ID | STRING | 每个网格单元的唯一标识符。 |
| **i_index** | 极坐标 I | INT | 模型网格的横向坐标索引。 |
| **j_index** | 极坐标 J | INT | 模型网格的纵向坐标索引。 |
| **geometry_wkt** | **地理多边形** | STRING (WKT) | 格式为 `POLYGON((...))`。用于 GIS 软件或 SQL 空间查询 (`ST_Contains`)，以确定某个坐标点是否落入该网格。 |
| **topography** | **地形海拔** | INT | 该网格单元的平均地面海拔高度（单位：英尺）。基准面通常为 NAVD88。 |

### 2.2 风暴潮水位预估 (Surge Elevation)

以下字段表示在特定飓风情境下，海水表面可能达到的**最高海拔高度**（单位：英尺）。
*   **数据清洗注记**：原始数据中的 `99.9`（表示未被淹没/干旱）已被转换为 `NULL`。

| 字段名称 (Feature) | 飓风等级 (Category) | 潮汐情景 (Tide) | 业务含义 |
| :--- | :--- | :--- | :--- |
| **c0_mean** | 热带风暴 (TS) | 平均潮 | 热带风暴在平均潮位时的最高水位。 |
| **c0_high** | 热带风暴 (TS) | **高潮** | 热带风暴在高潮位时的最高水位（风险更高）。 |
| **c1_mean / c1_high** | **1 级飓风** | Mean / High | Saffir-Simpson 1 级飓风下的最坏水位封顶值。 |
| **c2_mean / c2_high** | **2 级飓风** | Mean / High | 2 级飓风下的最坏水位封顶值。 |
| **c3_mean / c3_high** | **3 级飓风** | Mean / High | 3 级飓风（主要飓风）下的最坏水位封顶值。 |
| **c4_mean / c4_high** | **4 级飓风** | Mean / High | 4 级飓风下的最坏水位封顶值。 |
| **c5_mean / c5_high** | **5 级飓风** | Mean / High | 5 级飓风（灾难性）下的最坏水位封顶值。 |

## 3. 使用建议

1.  **最坏情况分析**：在做防灾规划或保险风控时，通常优先使用 `_high` 后缀的字段（如 `c3_high`），因为它考虑了高潮位叠加风暴潮的最不利组合。
2.  **空间关联**：由于这是网格数据，要查询具体建筑物的风险，需将建筑物的经纬度（Point）与 `geometry_wkt`（Polygon）进行空间连接（Spatial Join）。
3.  **单位**：所有高度数据的垂直单位均为 **英尺 (Feet)**。
